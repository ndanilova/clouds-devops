## Разберем по отдельности каждый продукт

### ***App Center*** 
 Azure App Center (полное название — Visual Studio App Center) — инструмент от Microsoft для непрерывного развёртывания и интеграции. С его помощью можно создавать приложения для iOS, Android и Windows, а также проводить автоматизированное тестирование. 

Некоторые возможности сервиса:

* **App Center Build.** Позволяет автоматически собирать мобильное приложение после каждого толчка в репозитории. 
* **App Center Test.** Даёт возможность проводить тесты интерфейса на реальном устройстве после успешной сборки.
* **App Center Distribute.** Позволяет распространять сборки приложения для бета-тестеров и публичных магазинов приложений.
* **App Center Diagnostics.** Позволяет собирать диагностические данные в реальном времени. 
* **App Center Analytics.** Даёт возможность отслеживать модели использования, активность пользователей и другие метрики. 
App Center можно подключать к GitHub, Bitbucket, GitLab или Azure DevOps.



### **App Configuration** 
это сервис для централизованного управления настройками и флагами функций в облаке. Он позволяет отделять конфигурации от кода, что способствует масштабируемости и быстрой итерации приложения.

Некоторые особенности Azure App Configuration:

* **Версиярование.** Каждая пара ключ-значение имеет версию, что даёт возможность откатывать или аудировать изменения.

* **Encryption.** Чувствительная информация шифруется на месте и в пути. 

* **Динамическая адаптация.** При изменении конфигураций не требуется переразвёртывание приложения. 

* **Управление конфигурацией для разных сред и географических областей.** Для этого можно использовать теги и метки.

* **Контроль доступности функций в реальном времени.**


### **App Service** 
  это служба для быстрого и лёгкого создания корпоративных веб-приложений и мобильных приложений для любых платформ и устройств и их развёртывания в масштабируемой и надёжной облачной инфраструктуре.


### **Application Insights**
это сервис для управления производительностью приложений в рамках платформы Azure Monitor. Он позволяет разработчикам и командам DevOps отслеживать и оптимизировать производительность и надёжность своих приложений. 

Некоторые возможности сервиса:

* **Мониторинг производительности в реальном времени.** 

* **Анализ поведения пользователей.** 

* **Проактивные оповещения.** Можно настроить интеллектуальные оповещения, которые уведомляют о аномалиях или проблемах с производительностью до того, как они повлияют на пользователей. 

* **Интеграция с DevOps.** 


### **Automation**
это облачный сервис, предоставляемый Microsoft Azure, который позволяет автоматизировать создание, развёртывание, мониторинг и обслуживание ресурсов в облаке Azure. 

Некоторые возможности Azure Automation:

* **Создание рунбукков.** Это последовательности автоматизированных задач и процессов, которые можно запланировать или запустить в ответ на конкретные события или условия. Рунбукки могут быть написаны на различных языках, включая PowerShell, Python и JavaScript. 

* **Управление обновлениями.** Позволяет управлять обновлениями для Windows и Linux-машин (виртуальных или физических) в Azure, наpremises средах и у других облачных провайдеров. 

* **Резервное копирование и восстановление.** Автоматизирует резервное копирование и восстановление данных и приложений в Azure. 

* **Мониторинг и оповещение.** Автоматизирует мониторинг и оповещение ресурсов Azure. Включает в себя сбор метрик производительности, анализ логов и отправку уведомлений при обнаружении проблем. 

* **Интеграция с системой source control.** Позволяет проверять рунбики или конфигурации в системе source control. 



### **Data Services** 
это набор сервисов для работы с данными в облачной платформе Azure.



### **SQL Data Warehouse**
это облачное, полностью управляемое решение для обработки и анализа больших объёмов данных от Microsoft Azure.



### **Azure Bastion**
это полностью управляемая служба в Microsoft Azure, которая обеспечивает безопасный и удобный удалённый доступ к виртуальным машинам через портал Azure.

Она позволяет подключаться к виртуальным машинам с помощью RDP и SSH без риска, который представляет использование общедоступных IP-адресов. 


### **Bandwidth** 
это пропускная способность для передачи данных в и из центров обработки данных Azure или других центров.



### **Azure Firewall**
это управляемая облачная служба для обеспечения безопасности сети, которая защищает ресурсы виртуальной сети Azure. Она позволяет создавать, применять и регистрировать политики подключения для приложений и сетей в подписках и виртуальных сетях.



### **Notification Hubs**
это полностью управляемый сервис push-уведомлений в Microsoft Azure. Он упрощает отправку персонализированных уведомлений пользователям на разных платформах и устройствах.



### **Power BI**
Azure и Power BI можно использовать вместе, чтобы визуализировать данные Azure, генерировать отчёты и панели управления, а также получать информацию в реальном времени.

Некоторые возможности интеграции:

* **Подключение к различным службам Azure.** Например, к Azure SQL Database, Azure Data Lake Storage, Azure Cosmos DB. 

* **Использование продвинутых функций аналитики.** Например, Azure Machine Learning и Azure Cognitive Services. Эти инструменты помогают получать ценную информацию из данных, проводить предиктивный анализ и реализовывать решения на основе искусственного интеллекта. 

* **Подключение Azure Analysis Services.**  Это полностью управляемая аналитическая служба в облаке, которая позволяет анализировать, моделировать и визуализировать данные. Использование Azure Analysis Services в Power BI позволяет создавать сложные отчёты и панели управления, интегрируя данные из различных источников и применяя разнообразные модели анализа. 



### **Recovery Services**
это сервис, который позволяет координировать переключение и репликацию виртуальных и физических машин в облачной платформе Azure.



### **Virtual Machines**
это сервис от Microsoft Azure, который позволяет создавать и управлять виртуальными машинами в облаке. 

VMs — это виртуальные экземпляры компьютерных систем, которые эмулируют поведение физических машин. Каждая VM запускает свои операционные системы и приложения, разделяя аппаратные ресурсы физического хоста.
